## Azure CDN (Recommended)

Azure Content Delivery Network helps in improving performance, availability, and security by using a distributed network of servers that can store cached content in point-of-presence locations, close to end users. Learn More

https://azure.microsoft.com/en-in/products/cdn/

## Azure Front Door

Azure Front Door (AFD) provides dynamic site acceleration that reduces response times while also allowing content delivery by caching at nearest edge servers for faster media downloads. Learn More

https://azure.microsoft.com/en-in/products/frontdoor/

![alt text](./assets/image-7.png)


## Azure Blob Storage (Recommended)

Azure Blob Storage allows you to store and access images, videos and other files. This effectively reduces the load on your web server thereby improving performance and user experience. Learn More

https://azure.microsoft.com/en-in/products/storage/blobs/

## Virtual Network

Virtual networks are logically isolated from each other in Azure. You can configure their IP address ranges, subnets, route tables, gateways, and security settings, much like a traditional network in your data center. Virtual machines in the same virtual network can access each other by default. Learn More

https://learn.microsoft.com/en-us/azure/virtual-network/virtual-networks-overview

# Create Static Web App

App Service Static Web Apps is a streamlined, highly efficient solution to take your static app from source code to global high availability. Pre-rendered content is distributed globally with no web servers required. Learn more

https://learn.microsoft.com/en-us/azure/static-web-apps/

![alt text](./assets/image-20.png)

![alt text](./assets/image-21.png)

![alt text](./assets/image-22.png)


Azure Static Web Apps offers the ability to use GitHub Identity or Azure Access Token to securely deploy your code to the platform. Static Web Apps also offers the ability to use a deployment token. Learn more

https://learn.microsoft.com/en-us/azure/static-web-apps/overview

![alt text](./assets/image-23.png)

Azure Static Web Apps provide global high availability for your applications' production environment. Static Web Apps also comes with the option to have a managed Azure Functions API. Managed functions and non-production environments for your Static Web Apps resource are regional. Learn more

https://learn.microsoft.com/en-us/azure/static-web-apps/

Distributed Functions (preview)
Backend functions will be dynamically deployed to the region closest to your end users for reduced network latency. Learn more

https://learn.microsoft.com/en-us/azure/static-web-apps/apis-overview
