# Microsoft Certified: Azure Developer Associate

Microsoft revises skills occasionally, but the standard breakdown:

# ✅ **1. Certification Overview**

**Certification Name:** Microsoft Certified: Azure Developer Associate  
**Exam Code:** **AZ-204: Developing Solutions for Microsoft Azure**  
**Target Role:** Azure Developer, Cloud Engineer, Backend Engineer (Cloud), DevOps Engineer (developer-focused)  
**Difficulty:** Intermediate  
**Prerequisite:** No mandatory prerequisites (good to have 1–2 years of cloud/backend experience)

----------

# ✅ **2. Skills Measured (Updated Outline)**

Microsoft revises skills occasionally, but the standard breakdown:

### **1. Develop Azure compute solutions (20–25%)**

-   Azure App Service
    
-   Azure Functions (HTTP triggers, Timer triggers, Queue triggers, Durable Functions)
    
-   Container-based apps (ACI, AKS)
    
-   Azure WebJobs
    

### **2. Develop for Azure Storage (15–20%)**

-   Blob Storage
    
-   Azure Files
    
-   Cosmos DB
    
-   Table Storage
    
-   Data redundancy (LRS/ZRS/GZRS)
    
-   Use AKV for secrets
    

### **3. Implement Azure security (20–25%)**

-   Manage identities with Azure AD
    
-   Implement OAuth2, Managed Identities
    
-   App configuration and Key Vault
    
-   Role-based access control (RBAC)
    

### **4. Monitor, troubleshoot, and optimize Azure solutions (15–20%)**

-   Application Insights
    
-   Log Analytics
    
-   Kusto Query Language (KQL)
    
-   Health monitoring + Alerts
    
-   Performance tuning
    

### **5. Connect to and consume Azure services (15–20%)**

-   Event Grid
    
-   Event Hub
    
-   Service Bus (Queues/Topics)
    
-   API Management
    
-   Message serialization (JSON/AVRO)
    

----------

# ✅ **3. Detailed Syllabus (Deep Breakdown)**

## ✅ Compute

-   App Service Plans, Deployment Slots
    
-   Background Services
    
-   Container images using ACR
    
-   AKS deployments + YAML
    
-   Durable Functions: Orchestrators + Activities
    

## ✅ Storage

-   Blob lifecycle management + tiering
    
-   Cosmos DB consistency levels (Strong/Eventual, Bounded Staleness)
    
-   Partitioning + RU/s configuration
    
-   Table Storage queries
    
-   Using Managed Identities for storage access
    

## ✅ Security

-   OAuth + OpenID Connect flow
    
-   Using Microsoft Identity Platform (MSAL)
    
-   Middleware authentication in .NET
    
-   Secrets rotation in AKV
    

## ✅ Monitoring

-   Telemetry collection
    
-   Distributed tracing
    
-   Log Analytics queries
    
-   Alerts (metric/Log-based)
    

## ✅ Service Integration

-   Event Grid publisher/subscriber
    
-   Event Hub consumer groups
    
-   Service Bus: sessions, dead-letter queues
    

----------

# ✅ **4. Learning Path (Step by Step)**

This is the optimal path to master everything **quickly + correctly**:

----------

## **Step 1: Learn Azure Fundamentals**

If you already know basics → skip  
Else cover:

-   Resource groups
    
-   VNets
    
-   Identity
    
-   Cost management
    

----------

## **Step 2: Learn Azure App Development (Core)**

-   Build .NET/Node/Java apps for Azure
    
-   Deploy to App Service
    
-   CI/CD with GitHub Actions or Azure DevOps
    

----------

## **Step 3: Learn Azure Serverless**

This is the **heart** of AZ-204:

✅ Azure Functions  
✅ Durable Functions  
✅ Bindings + Triggers  
✅ Logic Apps (optional)

----------

## **Step 4: Learn Azure Storage Deeply**

-   Blob container SAS tokens
    
-   Queues + dead-lettering
    
-   Cosmos DB (Core SQL API, NoSQL)
    
-   Change Feed
    

----------

## **Step 5: Learn Messaging & Eventing**

-   When to use Event Grid?
    
-   When to use Service Bus?
    
-   When to use Event Hub?
    
-   Retry pattern
    
-   Poison message handling
    

----------

## **Step 6: Security**

-   Azure AD App registrations
    
-   Managed identities
    
-   RBAC
    
-   App Config + Key Vault
    

----------

## **Step 7: Monitoring**

-   Application Insights setup
    
-   Dependency tracking
    
-   Availability tests
    
-   KQL querying
    

----------

## **Step 8: Final Revision**

-   Microsoft Learn modules
    
-   Official practice test
